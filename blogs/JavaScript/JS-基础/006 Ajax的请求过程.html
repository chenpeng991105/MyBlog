# Ajax的请求过程

Ajax（Asynchronous Javascript And XML）是一种异步请求数据的技术，可以改善用户的体验和提升程序的性能

请求过程：

  1. 创建一个异步请求对象
  2. 创建一个 HTTP 请求，并指定该 HTTP 请求的类型，URL 以及验证信息
  3. 发送请求
  4. 设置响应 HTTP 请求状态变化的函数
  5. 获取异步请求返回的数据

get 请求

function ajax(url) {
    let xhr
    if(window.XMLHttpRequest){
        xhr = new XMLHttpRequest()
    }else{
        xhr = new ActiveXObject('Microsoft.XMLHTTP')// IE异步请求对象
    }
    xhr.onreadystatechange = function() {
        if (xhr.readyState === 4) {
            if ((xhr.status >= 200 && xhr.status < 300) || xhr.status == 304) {
                console.log(JSON.parse(xhr.responseText))
            }
        }
    }
    xhr.open('GET', 'url', true)// true代表异步，默认为true
    xhr.send(null)
}

参数说明：

open(method, url, async)

  • method：请求类型，GET 或 POST
  • url：文件在服务器上的位置，相对位置或绝对位置
  • async：true（异步）或 false（同步），true表示脚本会在 send 方法之后继续执行，不用等待服务器的响应

服务器响应数据：

  • responseText：字符串形式的响应数据
  • responseXML：XML 形式的响应数据

onreadystatechange：接收到服务器响应时触发

readyState：请求状态

  • 0：未初始化，还没调用 open() 方法
  • 1：已打开，已调用 open() 方法，还没调用 send() 方法
  • 2：已发送，已调用 send() 方法，还没收到响应
  • 3：接收中，已经收到部分响应
  • 4：请求已完成，已经收到所用响应，可以使用了

status：响应状态码

  • 200：请求成功
  • 304：该资源在上次请求没有任何修改（通常用于浏览器的缓存机制）
  • 400：请求出现语法错误
  • 403：服务器拒绝请求
  • 404：服务器找不到资源
  • 500：服务器内部错误
  • 503：服务器由于维护或过载未能响应

promise封装：

const p = new Promise((resolve, reject) => {
    let xhr = new XMLHttpRequest()
    xhr.onreadystatechange = function () {
        if (xhr.readyState === 4) {
            if ((xhr.status >= 200 && xhr.status < 300) || xhr.status == 304)) {
                resolve(JSON.parse(xhr.responseText))
            } else {
                reject('error')
            }
        }
    }
    xhr.open('get', 'url')
    xhr.send(null)
})
p.then(value => {
    console.log(value)
}).catch(reason => {
    console.log(reason)
})
