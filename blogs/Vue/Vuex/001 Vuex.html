# 概述

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式，它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化

Vuex 运用了单例模式，不管我们尝试创建多少次，它都只给你返回第一次所创建的唯一的一个实例

# Vuex 工作流程

# Vuex 目录结构

store：

  • index.js：初始化 store
  • state.js：store 中管理的状态
  • getter.js：对状态进行加工处理
  • mutations.js：修改 store 中的状态
  • action.js：异步提交 mutation 修改状态
  • mutations-type.js：存放 mutations 方法常量的文件，可按需加入
  • modules：模块化 store

# Vuex 详解

# 初步使用 Vuex

1.初始化 store

index.js

import Vue from 'vue'
import Vuex from 'vuex'

//挂载Vuex
Vue.use(Vuex)

//创建VueX对象
const store = new Vuex.Store({
    state:{
        //存放的键值对就是store中管理的状态
        name: 'hello Vuex'
    }
})
export default store

# modules

Vuex 允许我们将 store 分割成模块（module）。每个模块拥有自己的 state、mutation、action、getter、甚至是嵌套子模块——从上至下进行同样方式的分割：

const moduleA = {
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... },
  getters: { ... }
}

const moduleB = {
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... }
}

const store = new Vuex.Store({
  modules: {
    a: moduleA,
    b: moduleB
  }
})

store.state.a // -> moduleA 的状态
store.state.b // -> moduleB 的状态
