Vue响应式数据

chenpeng 2021-05-07 Vue

# 1.Vue 是如何实现响应式数据的？

Vue 是采用数据劫持结合发布订阅模式来实现响应式数据的，具体是通过以下4个步骤：

  1. observer(监听器)：对数据对象进行遍历，包括子属性对象的属性，利用 Object.defineProperty 给每个属性都加上 set 和 get 方法，这样当给对象的某个属性赋值时，就会触发 set 方法，就能监听到数据的变化。
  2. compile(解析器)：解析模板指令，将指令的变量替换成数据，然后初始化渲染页面视图，并为每个指令对应的节点绑定更新函数，添加监听数据的订阅者，一旦数据有变动，就会收到通知，调用更新函数更新数据。
  3. watcher(订阅者 )：watcher 是 observer 和 compile 之间通信的桥梁，主要任务是订阅 observer 中的属性值变化的消息，watcher 在自身实例化时会向 dep 中添加自己，它有一个 update 方法，当收到属性值变化的消息时，触发 compile 中绑定的更新函数。
  4. Dep(订阅器)：Dep 采用发布订阅模式。当页面使用对应属性时，会通过 Dep 进行依赖收集（收集当前组件的 watcher），如果属性值发生变化，会触发 Dep 的 notify 方法，notify 方法会触发当前收集的组件的 watcher 的 update 方法。

# 2.Vue 如何实现对数组的监听？

Vue 重写了数组的以下原型方法：

  • push
  • pop
  • shift
  • unshift
  • reverse
  • sort
  • splice

Vue 将 data 中的数组，进行了原型链的重写，指向了自己定义的数组原型，当调用数组的上述方法时，可以通过依赖更新，如果数组中包含着引用类型数据，会对数组中的引用类型数据再次进行监听。

# 3.直接给数组项赋值，Vue 能检测到变化吗？

由于 JavaScript 的限制，Vue 不能检测到数组以下的变动：

  • 当利用索引直接设置一个数组项时，例如 vm.items[indexOfItem] = newValue
  • 当修改数组的长度时，例如 vm.items.length = newLength

对于第一个问题，Vue 提供了以下操作方式：

// Vue.set
Vue.set(vm.items, indexOfItem, newValue)

// vm.$set，Vue.set的一个别名
vm.$set(vm.items, indexOfItem, newValue)

// Array.prototype.splice
vm.items.splice(indexOfItem, 1, newValue)

# 4.Vue如何通过 vm.$set() 来解决对象/数组新增不能响应的问题？

Vue 提供了 Vue.set(object, propertyName, value) 和 Vue.set(vm.items, indexOfItem, newValue) 来解决给对象新增属性和给数组新增元素不能触发响应式的问题。

其原理如下：

  • 如果目标是数组，直接使用数组的 splice 方法触发响应式。
  • 如果目标是对象，会先判断对象属性是否存在，对象是否是响应式，如果要对属性进行响应式处理，则是通过 defineReactive 方法进行响应式处理。

# 5.Object.defineProperty 有什么缺点？（为什么 Vue3.0 开始使用 Proxy 实现响应式）

  • Object.defineProperty 只能劫持对象的属性，因此需要遍历对象的每个属性，而 Proxy 可以直接代理对象。
  • Object.defineProperty 对新增属性需要手动进行观察，由于 Object.defineProperty 劫持的是对象的属性（第一点），所以新增属性时，需要重新遍历对象，对其新增属性再使用 Object.defineProperty 进行劫持 （正是这个原因导致我们在给 data 中的数组或对象新增属性时，需要使用 $set 才能保证视图可以更新）。
  • Proxy 性能高，支持13种拦截方式。
